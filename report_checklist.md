# Roof Inspection Report Checklist

Use this checklist before delivering a roof inspection file.

## 1. Property and inspection metadata

- Property address, city, state, and postal code are present.
- Inspection date is present.
- Inspector name and organization are present.
- Inspection limitations are disclosed.
- Access method is documented.

## 2. Roof system description

- Primary roof covering is identified.
- Approximate roof age is documented when known.
- Slope, drainage, and roof plane notes are included.
- Penetrations, flashings, valleys, ridge, vents, gutters, and accessories are reviewed when visible.

## 3. Photo documentation

- Overview photos show each accessible elevation and roof plane.
- Close-up photos are paired with context photos.
- Photo filenames are referenced in the photo manifest.
- Each material observation references one or more photo IDs when possible.

## 4. Damage and condition labels

- Observations use the public damage label taxonomy.
- Severity is documented on a 0-5 scale.
- Probable cause is separated from the visible condition.
- Maintenance, age-related wear, installation issues, and storm-related conditions are distinguished.

## 5. Recommendations

- Recommendations are tied to specific observations.
- Urgent items are separated from routine maintenance.
- Repair vs. replacement logic is stated plainly.
- Further evaluation needs are documented.

## 6. Final review

- The JSON file is complete enough to score cleanly.
- The Markdown report is readable without the JSON file.
- The file contains no private homeowner information beyond what is necessary for the inspection record.
- Disclaimers are accurate and not overbroad.
