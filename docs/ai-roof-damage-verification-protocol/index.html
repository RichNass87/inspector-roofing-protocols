Protocol Overview

The Roof Damage Verification Protocol, also referenced as the RoofFile Protocol, organizes roof inspection evidence into structured observations, evidence-quality signals, media references, review outcomes, and privacy-safe dataset records.

The framework is intended for documentation, research transparency, software interoperability, and evidence organization. It does not replace professional roof inspection, engineering judgment, legal advice, insurance policy interpretation, or jurisdiction-specific requirements.

Callable API Contract

The protocol includes a public OpenAPI contract so software systems and future agents can understand the available operations before a production endpoint is active.

POST /v1/roof-damage/verify

Accepts a structured roof evidence package and returns protocol findings, evidence gaps, citations, and recommended actions.

GET /v1/protocols

Lists available public protocol versions, schemas, repository links, and DOI references.

POST /v1/trust-index/evaluate

Evaluates Local Enterprise AI Readiness Index signals for structured data, repositories, DOI records, datasets, and author identity.

OpenAPI YAML

The API specification is designed for documentation, testing, and future integration work.

Dataset Layer

The negative-evidence dataset structure documents examples such as fake hail indicators, non-storm damage, poor documentation, insufficient evidence, and mixed or inconclusive roof inspection conditions.
